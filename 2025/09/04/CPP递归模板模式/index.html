CPP递归模板模式

|阅读量:

C++的CRTP

基本结构

  • 关键点在于派生类Derived继承自以Derived自身作为模板参数的基类模板Base。

主要用途

1. 静态多态（编译期多态）

在编译器确定调用关系，无需创建和使用虚函数

2. 将功能添加到派生类

  • 每个派生类有自己独立的计数器
  • 不需要修改派生类代码即可添加计数功能
  • 计数逻辑完全在编译期实现

3. 接口注入

SQL查询

使用CRTP,可以实现:

  • 避免虚函数开销：没有使用 virtual，所有调用在编译期解析，性能更高。
  • 链式调用支持：基类方法返回 Derived&，使得方法可以连续调用。
  • 代码复用：QueryBuilder 提供通用的 SQL 拼接逻辑，SqlQuery 只需实现 build()。

4. 标准库案例

安全的返回自己的智能指针

#include <memory>
#include <iostream>

class Good : public std::enable_shared_from_this<Good> {
public:
    std::shared_ptr<Good> getptr() {
        return shared_from_this();
    }
};

class Bad {
public:
    std::shared_ptr<Bad> getptr() {
        return std::shared_ptr<Bad>(this);  // 危险！
    }
    ~Bad() { std::cout << "Bad destroyed\n"; }
};

int main() {
    // 正确用法
    std::shared_ptr<Good> gp1 = std::make_shared<Good>();
    std::shared_ptr<Good> gp2 = gp1->getptr();
    std::cout << "gp2.use_count() = " << gp2.use_count() << "\n";  // 输出: 2

    // 错误用法会导致双重释放
    std::shared_ptr<Bad> bp1 = std::make_shared<Bad>();
    std::shared_ptr<Bad> bp2 = bp1->getptr();  // 运行时错误！
    std::cout << "bp2.use_count() = " << bp2.use_count() << "\n";  // 不会执行到这里
}

当需要从一个对象的成员函数中获取指向自身的shared_ptr时，如果直接return shared_ptr(this)会导致：

  • 新的shared_ptr不知道已有控制块的存在
  • 对象会被多个独立的引用计数管理最终导致双重释放

CRTP解决方案：

基类模板存储了一个weak_ptr，当首次创建shared_ptr时初始化这个weak_ptr，shared_from_this()通过weak_ptr安全地生成新的shared_ptr

关键限制：

  • 必须先有至少一个shared_ptr实例存在
  • 不能在构造函数中调用shared_from_this()
  • 继承时必须使用公有继承

典型应用场景：

  • 异步回调中保持对象生命周期
  • 观察者模式中的自引用
  • 需要将this传递给需要shared_ptr的API时
